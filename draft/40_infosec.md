# 40_infosec - 資安需求與權限矩陣

## 資料分級

- STR/SAR 原始文本：高度機密
- 模型輸出的結構化欄位與摘要：高度機密
- 模型訓練與測試資料集：高度機密

## 角色與權限（摘要）

| 角色              | 檢視原始 STR 文本 | 檢視模型摘要 | 編輯/確認摘要 | 檢視訓練樣本 | 檢視操作日誌 |
|-------------------|-------------------|--------------|--------------|--------------|--------------|
| 分析人員         | 是                | 是           | 是           | 否           | 僅限本人操作 |
| 洗錢防制主管     | 是                | 是           | 否           | 否           | 是           |
| 法遵人員         | 是                | 是           | 否           | 否           | 是           |
| 模型工程師       | 在遮罩後          | 在遮罩後     | 否           | 是           | 是           |
| 稽核單位         | 是                | 是           | 否           | 在遮罩後     | 是           |

說明：「在遮罩後」表示關鍵識別資訊（姓名、帳號、統編等）需經過假名化或部分遮蔽。

## 存取控制

- 所有 STR 相關系統僅允許透過內網或高安全性 VPN 存取。
- 必須使用個人名義帳號與多因子認證。
- 高敏感個案可設定額外限制（例如僅特定分析人員可見）。

## 日誌與監控

- 需記錄每一次開啟 STR 個案畫面、檢視模型輸出、修改與確認摘要的操作。
- 需能偵測異常存取（例如一名使用者短時間大量檢視非分行轄下個案）。

## 技術安全要求

- 模型服務部署在受控環境（不可直接對外暴露）。
- 禁止使用將資料送往外部第三方雲端 LLM 的架構（除非經過法遵與資安核准的特定方案）。
